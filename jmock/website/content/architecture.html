Architectural Overview

jMock has a layered architecture. An application's tests will usually use a high level API that provides a quick and convenient way to define expectations and stubs. That high level API is syntactic sugar for a flexible but verbose core API. The core API provides extensible framework for defining mock objects and expectations compositionally. The high-level and core APIs both make use of some fundamental interfaces — Verifiable, Constraint — and constraint classes. Constraints are used to specify acceptable argument values for expected method invocations.

Although it makes it easy to do most common tasks, a high level API cannot cover ever possible eventuality. The high level API therefore contains hooks with which a user can access the core API to add custom expectations and stubs if necessary.

High Level APIs

jMock ships with two high-level APIs: DynaMock, which will be familiar to users of the old mockobjects.com library, and Hot Mock, a new API designed to be expressive and easier to read than DynaMock. Other high level APIs can be implemented separately by using the core jMock framework. Third-party APIs that are built upon the jMock core may be announced on and linked from the jMock website.
