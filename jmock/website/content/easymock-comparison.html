Comparing jMock and EasyMock

EasyMock is also a dynamic mock object library for Java. What are the main differences between jMock and EasyMock?

EasyMock doesn't use strings to identify methods

EasyMock therefore better cooperates with refactoring tools. In practice we have found that this does not effect jMock badly, especially if you use tools like IntelliJ or Eclipse.

jMock makes it easier to keep tests flexible.

Constraints, matchers, etc.

jMock is designed for readability

jMock has a "specifiation" API. EasyMock uses a "record/playback" API. Tests written with jMock are much easier to read as documentation.

jMock lets you stub methods with side effects

See here.

jMock gives you more control over the order of invocations

jMock lets the programmer specify partial orders of invocations and specify ordering between invocations on different mock objects.
